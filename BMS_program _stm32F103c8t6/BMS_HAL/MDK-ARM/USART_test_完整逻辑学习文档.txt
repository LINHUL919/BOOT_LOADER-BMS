一、系统角色说明
================================================================================

本 BMS (Battery Management System) 程序运行在 STM32F103C8 微控制器上,
通过软件 I2C 总线连接 TI BQ76940 电池管理前端芯片, 实现对 15 串锂电池包
(三元锂 NMC, 标称容量 20Ah) 的完整管理。

核心功能:
  1) 电压采集: 15节单体电压逐节读取 + 滑动平均滤波 + 跳变去抖
  2) 温度采集: NTC 10K/B3380 热敏电阻, B值法计算
  3) 电流采集: BQ76940内部库仑计, 4mR采样电阻, 移动平均滤波
  4) SOC估算:  库仑积分 + OCV校准 + 满充校准 + 容量学习 (bms_soc.c)
  5) 保护控制: 过压/欠压/过流/过温/低温充电五重保护, 自动恢复
  6) 均衡控制: 被动均衡, 支持 bitmask 多路同时设置, 奇偶交替
  7) MOS控制:  充电MOS/放电MOS 独立或联合开关
  8) 串口CLI: USART1 115200, ASCII命令行, 支持status/chg/dsg/bal/mos等
  9) 看门狗:   IWDG 约6.5s超时, Idle钩子喂狗
 10) ALERT中断: BQ76940 PB2下降沿, xTaskNotify立即唤醒BMS_Task

操作系统: FreeRTOS V202212.01 (抢占式调度, 1ms Tick)
任务数: 2个应用任务 + 1个Idle任务
通信方式: USART1 (PA9/TX, PA10/RX) + DMA发送
I2C方式: 软件模拟 (PB8/SCL, PB9/SDA, 开漏)




二、内存 / 外设 / 存储 / 关键资源划分图
================================================================================

  STM32F103C8 资源分布
  +------------------------------------------------------------------+
  |  Flash: 64KB (0x0800_0000 ~ 0x0800_FFFF)                        |
  |  +--------------------------+-----------------------------------+|
  |  | 0x0800_0000 ~ 0x0800_FBFF| 程序代码 + 只读常量 (~63KB)      ||
  |  +--------------------------+-----------------------------------+|
  |  | 0x0800_FC00 ~ 0x0800_FFFF| SOC Flash存储页 (1KB)             ||
  |  |                          | magic(4)+soc_mAh(4)+full_cap(4)   ||
  |  |                          | +cycle_x10(2)+pad(2)+checksum(4)  ||
  |  |                          | = 20字节, 半字编程                ||
  |  +--------------------------+-----------------------------------+|
  +------------------------------------------------------------------+
  |  SRAM: 20KB (0x2000_0000 ~ 0x2000_4FFF)                         |
  |  +--------------------------+-----------------------------------+|
  |  | 栈区 (从顶部向下)       | 中断栈 + main栈                   ||
  |  +--------------------------+-----------------------------------+|
  |  | FreeRTOS堆: 10KB        | 任务栈+TCB+信号量+互斥量          ||
  |  |  configTOTAL_HEAP_SIZE  | BMS_Task栈=1KB, UART_Task栈=1KB  ||
  |  |                         | Idle栈=512B, Mutex+Sema~200B      ||
  |  +--------------------------+-----------------------------------+|
  |  | 全局变量区              | bms_data(~130B), s_soc(~48B)      ||
  |  |                         | g_prot(8B), 滤波缓冲(~400B)       ||
  |  |                         | UART缓冲(~50B), DMA描述(~60B)     ||
  |  +--------------------------+-----------------------------------+|
  +------------------------------------------------------------------+

  外设使用一览
  +------------------------------------------------------------------+
  | 外设     | 用途            | 引脚      | 配置                    |
  +----------+-----------------+-----------+-------------------------+
  | USART1   | 串口CLI         | PA9/PA10  | 115200,8N1, DMA TX      |
  | DMA1_CH4 | USART1_TX       | -         | M->P, Byte, Normal      |
  | TIM3     | HAL时基(SysTick)| -         | 1ms中断, 优先级=1       |
  | IWDG     | 独立看门狗      | -         | PSC=64, RLD=4095, ~6.5s |
  | EXTI2    | BQ76940 ALERT   | PB2       | 下降沿, 优先级=6        |
  | GPIOA8   | BQ唤醒脉冲      | PA8       | 推挽输出                |
  | GPIOB8/9 | SW I2C SCL/SDA  | PB8/PB9   | 开漏输出                |
  | SysTick  | FreeRTOS Tick   | -         | 1ms, 优先级=15          |
  | Flash    | SOC数据持久化   | -         | 最后1KB页               |
  +----------+-----------------+-----------+-------------------------+

  时钟树: HSE(8MHz) -> PLL x9 -> SYSCLK=72MHz
          AHB=72MHz, APB2=72MHz, APB1=36MHz, LSI=40KHz -> IWDG

四、完整调用树
================================================================================

  4.1 主启动链路 (从Reset到调度器)
  -----------------------------------------------------------------------

  Reset_Handler()                           [startup_stm32f103xb.s]
  └── main()                                [main.c]
      ├── HAL_Init()                        [HAL初始化, SysTick/Flash/NVIC]
      ├── SystemClock_Config()              [HSE+PLL -> 72MHz]
      ├── MX_GPIO_Init()                    [PA8推挽, PB2下降沿EXTI, PB8/9开漏]
      ├── MX_DMA_Init()                     [DMA1_CH4 for USART1_TX, 优先级6]
      ├── MX_USART1_UART_Init()             [115200, 8N1, TX+RX]
      ├── MX_IWDG_Init()                    [PSC=64, RLD=4095, ~6.5s超时]
      ├── SW_I2C_Init()                     [PB8/PB9开漏, SCL/SDA拉高]
      ├── BQ76940_Init()                    [BQ76940完整初始化]
      │   ├── BQ76940_CommHealthReset()     [清零通信统计]
      │   ├── PA8脉冲唤醒 (100ms高+200ms低) [SHIP->NORMAL模式]
      │   ├── 12个寄存器写入                [SYS_STAT/CELLBAL/CTRL/PROTECT/CC_CFG]
      │   │   ├── SYS_STAT  = 0xFF         [清除所有标志]
      │   │   ├── CELLBAL1/2/3 = 0x00      [关均衡]
      │   │   ├── SYS_CTRL1 = 0x18         [ADC_EN=1, TEMP_SEL=1]
      │   │   ├── SYS_CTRL2 = 0x43         [CC_EN=1, DSG=1, CHG=1]
      │   │   ├── PROTECT1/2/3 = 0x00/0x00/0x00
      │   │   ├── OV_TRIP/UV_TRIP = 0x00/0x00
      │   │   └── CC_CFG = 0x19            [库仑计配置]
      │   ├── BQ_ReadCalibration()          [读GAIN/OFFSET校准值]
      │   ├── 计算并写入OV_TRIP/UV_TRIP     [OVP=4400mV, UVP=2400mV]
      │   ├── PROTECT1=0xFF, PROTECT2=0xFF  [SCD/OCD最大保护]
      │   ├── SYS_STAT=0xFF                [清状态]
      │   ├── 初始化滤波缓冲               [cell_mv[]=3600, temp=25.0]
      │   └── BQ76940_ClearBalance()        [确保均衡关闭]
      ├── SOC_Init()                        [SOC模块初始化]
      │   ├── memset(&s_soc, 0, ...)        [清零状态]
      │   ├── 设默认值 (50%×20000mAh)       [full_cap=20000, soc=10000]
      │   ├── flash_load()                  [尝试从0x0800FC00恢复]
      │   │   ├── 检查magic==0x534F4321     ["SOC!"]
      │   │   ├── 校验checksum              [半字异或^0xA5A5A5A5]
      │   │   └── 合理性: soc>=0, full_cap>=1000
      │   ├── 限幅 + 计算soc_percent        [0~100%]
      │   └── 计算soh_percent               [full_cap/nominal×100]
      ├── UART_StartReceive()               [启动USART1中断接收]
      ├── UART_DMA_Init()                   [创建DMA完成信号量]
      ├── xSemaphoreCreateMutex()           [创建I2C互斥量: xI2CMutex]
      ├── xTaskCreate(BMS_Task, pri=3)      [电池采集+保护任务, 栈256字]
      ├── xTaskCreate(UART_Task, pri=2)     [串口命令处理任务, 栈256字]
      ├── BQ76940_SetAlertTask(xBmsTaskHandle) [注册ALERT通知目标]
      └── vTaskStartScheduler()             [启动FreeRTOS, 不返回]

  4.2 BMS_Task 周期执行链路 (优先级3, 500ms/ALERT唤醒)
  -----------------------------------------------------------------------

  BMS_Task()                                [main.c, 周期500ms]
  └── for(;;)
      ├── ulTaskNotifyTake(500ms)           [阻塞: 500ms超时或ALERT唤醒]
      ├── xSemaphoreTake(xI2CMutex)         [获取I2C总线]
      │
      ├── [ALERT分支] if(g_bq_alert_flag)
      │   ├── g_bq_alert_flag = 0           [清标志]
      │   ├── BQ76940_ReadAlerts()          [读SYS_STAT: UV/OV/SCD/OCD]
      │   └── BQ76940_ProtectionCheck()     [保护判断+MOS控制]
      │
      ├── [周期分支] BQ76940_CommCheck()    [通信健康检查]
      │   ├── 连续失败<5: return 1(正常)
      │   ├── 连续失败>=5 且 间隔>=3s:
      │   │   └── BQ76940_Init()            [完整重初始化, return 0]
      │   └── 连续失败>=5 且 间隔<3s: return 0(等待)
      │
      ├── BQ76940_UpdateAll()               [一次性采集全部数据]
      │   ├── BQ76940_ReadAllCells()        [15节电压采集]
      │   │   ├── 暂停均衡(如有)            [避免均衡电流影响采样]
      │   │   ├── for i=0~14:
      │   │   │   ├── BQ_ReadReg(0x0C+i*2)  [读HI字节]
      │   │   │   ├── BQ_ReadReg(0x0D+i*2)  [读LO字节]
      │   │   │   ├── sample = raw*gain/1000+offset
      │   │   │   └── BQ_FilterAndUpdateCell(i, sample)
      │   │   │       ├── 越界检查(2000~4500mV)
      │   │   │       ├── 跳变限幅(>30mV连续3次才接受)
      │   │   │       └── BQ_CellFilterUpdate() [4点滑动平均+10mV去抖]
      │   │   ├── pack_mv = sum(cell_mv[])
      │   │   └── 恢复均衡(如之前暂停)
      │   ├── BQ76940_ReadTemp()            [温度采集]
      │   │   ├── 读TS1_HI/LO (0x2C/0x2D)
      │   │   ├── vtsx = raw*382/1000       [ADC->mV]
      │   │   ├── Rt = 10000*vtsx/(3300-vtsx) [分压->NTC阻值]
      │   │   ├── T = 1/(1/T2+ln(Rt/Rp)/B)-273.15 [B值法]
      │   │   ├── 越界检查(-40~120°C)
      │   │   └── BQ_TempFilterUpdate()     [4点滑动平均+0.3°C去抖]
      │   ├── BQ76940_ReadCurrent()         [电流采集]
      │   │   ├── 检查CC_READY(SYS_STAT bit7)
      │   │   ├── 读CC_HI/LO (0x32/0x33)
      │   │   ├── raw<=0x7D00: I=raw*2.11   [放电,正]
      │   │   ├── raw>0x7D00: I=-(0xFFFF-raw)*2.11 [充电,负]
      │   │   ├── 4点移动平均滤波
      │   │   └── ±3mA 死区消除零漂
      │   └── BQ_ReadMosStatus()            [读SYS_CTRL2更新chg_on/dsg_on]
      │
      ├── BQ76940_ProtectionCheck()         [保护判断]
      │   ├── BQ_CheckOV()                  [过压: >4200mV关CHG+清均衡]
      │   ├── BQ_CheckUV()                  [欠压: <2800mV关DSG+清均衡]
      │   ├── BQ_CheckOC()                  [过流: >5000mA关全部+清均衡]
      │   ├── BQ_CheckOT()                  [过温: >60°C关全部+清均衡]
      │   └── BQ_CheckLT()                  [低温: <0°C关CHG+清均衡]
      │
      ├── BQ76940_ReadAlerts()              [读硬件报警标志]
      │
      ├── SOC_Update(current, pack, avg, chg_on, dt) [SOC库仑计更新]
      │   ├── 库仑积分: soc_mAh -= I*dt/3600000
      │   ├── 循环计数: 充电累积>=20000mAh -> +1循环
      │   ├── OCV校准: 静置30s -> 电压查表渐进10%修正
      │   ├── 满充校准: V>=4150 & |I|<200 -> SOC=100%
      │   ├── 空电检测: V<=2800 -> SOC=0%
      │   ├── 容量学习: 空->满, EMA 80%旧+20%新
      │   ├── 限幅 + 计算百分比
      │   └── 定时Flash保存: 60s检查, 变化>=1%才擦写
      │
      └── xSemaphoreGive(xI2CMutex)         [释放I2C总线]

  4.3 UART_Task 命令处理链路 (优先级2, 50ms)
  -----------------------------------------------------------------------

  UART_Task()                               [main.c, 周期50ms]
  └── for(;;)
      ├── xSemaphoreTake(xI2CMutex)         [获取I2C总线]
      ├── UART_CmdProcess()                 [检查s_cmd_ready标志]
      │   └── BMS_ParseCmd(s_rx_line)       [解析ASCII命令]
      │       ├── "status"  -> BQ76940_PrintData()  [打印全部数据]
      │       ├── "chg on"  -> BQ76940_OpenCHG()    [开充电MOS]
      │       ├── "chg off" -> BQ76940_CloseCHG()   [关充电MOS]
      │       ├── "dsg on"  -> BQ76940_OpenDSG()    [开放电MOS]
      │       ├── "dsg off" -> BQ76940_CloseDSG()   [关放电MOS]
      │       ├── "mos on"  -> BQ76940_OpenAll()     [全开]
      │       ├── "mos off" -> BQ76940_CloseAll()    [全关]
      │       ├── "bal off"       -> ClearBalance()  [关均衡]
      │       ├── "bal odd"       -> SetBalance(0x5555) [奇数单体]
      │       ├── "bal even"      -> SetBalance(0x2AAA) [偶数单体]
      │       ├── "bal status"    -> GetBalanceMask()   [查询]
      │       ├── "bal mask 0xNN" -> SetBalance(NN)     [自定义]
      │       └── "bal N"         -> SetBalance(1<<(N-1)) [单节]
      ├── xSemaphoreGive(xI2CMutex)         [释放I2C总线]
      └── vTaskDelay(50ms)

            |   uv_flag=1 且全部>3000mV? (200mV回差)
            |          -> OpenDSG(), uv_flag=0
            |
    +-------v-------+     +------------------+
    | BQ_CheckOC()  +---->| |I|>5000mA?      |
    +-------+-------+  是 |  -> CloseAll()   |
            |   否        |  -> ClearBalance()|
            |             |  -> oc_flag=1     |
            |             +--------+---------+
            |                      |
            |   oc_flag=1 且 |I|<4500mA? (500mA回差)
            |          -> OpenAll(), oc_flag=0
            |
    +-------v-------+     +------------------+
    | BQ_CheckOT()  +---->| temp>60°C?       |
    +-------+-------+  是 |  -> CloseAll()   |
            |   否        |  -> ClearBalance()|
            |             |  -> ot_flag=1     |
            |             +--------+---------+
            |                      |
            |   ot_flag=1 且 temp<55°C? (5°C回差)
            |          -> OpenAll(), ot_flag=0
            |
    +-------v-------+     +------------------+
    | BQ_CheckLT()  +---->| temp<0°C?        |
    +-------+-------+  是 |  -> CloseCHG()   |
            |   否        |  -> ClearBalance()|
            |             |  -> lt_flag=1     |
            |             +--------+---------+
            |                      |
            |   lt_flag=1 且 temp>5°C? (5°C回差)
            |          -> OpenCHG(), lt_flag=0
            |
            v
    +=======+=======+
    |    返回        |
    +================+

  保护策略总结:
    - 本工程采用 [自动恢复] 策略
    - 过压: 关充电, 全部低于恢复阈值后自动重新开充电
    - 欠压: 关放电, 全部高于恢复阈值后自动重新开放电
    - 过流: 关全部MOS, 电流降低后自动恢复
    - 过温: 关全部MOS, 温度降到55°C以下自动恢复
    - 低温: 关充电MOS(放电不受影响), 温度升到5°C以上自动恢复
    - 所有保护触发时同步清除均衡
    - 无 [锁死故障] 或 [手动恢复] 模式


  5.4 SOC更新流程
  -----------------------------------------------------------------------

    +================+
    | SOC_Update()   |  <-- 每500ms调用
    +========+=======+
             |
    +--------v--------+
    | 1.库仑积分       |  soc_mAh -= I * dt / 3600000
    +--------+--------+
             |
    +--------v--------+
    | 2.循环计数       |  充电累积 >= 20000mAh -> +1循环
    +--------+--------+
             |
    +--------v-----------+
    | 3.OCV校准          |  |I|<50mA 持续30s?
    +--+---------+-------+  -> 电压查表, 渐进10%修正
     否|         |是
       |   +-----v------+
       |   | 查OCV表     |
       |   | soc += 10%差|
       |   +-----+------+
       |         |
    +--v---------v--------+
    | 4.满充检测           |  chg_on && |I|<200 && V>=4150?
    +--+---------+--------+  -> soc=100%, 容量学习(如armed)
     否|         |是
       |   +-----v---------+
       |   | soc=full_cap   |
       |   | EMA更新full_cap|
       |   +-----+---------+
       |         |
    +--v---------v--------+
    | 5.空电检测           |  V_cell<=2800?
    +--+---------+--------+  -> soc=0%, arm学习
     否|         |是
       |         |
    +--v---------v--------+
    | 6.限幅+百分比计算    |
    +--------+------------+
             |
    +--------v-----------+
    | 7.定时Flash保存     |  60s检查, 变化>=1%才写入
    +--------------------+

八、实际使用步骤 (最小可跑通步骤)
================================================================================

  步骤1: 编译与下载
  -----------------------------------------------------------------------
  - 打开 MDK-ARM/USART_test.uvprojx (Keil uVision 5)
  - 编译器: ARMCC v5.06
  - 目标芯片: STM32F103C8 (Medium-density)
  - 编译 -> 下载 (ST-Link / J-Link)
  - 注意: Flash最后1KB(0x0800FC00)被SOC模块使用, 不要覆盖

  步骤2: 上电启动
  -----------------------------------------------------------------------
  - 连接串口: PA9(TX)→USB-TTL RX, PA10(RX)→USB-TTL TX, GND共地
  - 波特率: 115200, 8N1, 无流控
  - 上电后观察串口是否有输出(首次需发送命令触发)
  - 确认: BQ76940_Init() 成功 → PA8脉冲唤醒 → I2C通信正常
  - 优先看: main.c → BQ76940_Init() → BQ_ReadCalibration()
  - 关键变量: bms_data.adc_gain (正常应~365~395), bms_data.adc_offset

  步骤3: 检查采样是否正常
  -----------------------------------------------------------------------
  - 串口发送: status + 回车
  - 观察输出:
      Cell01~Cell15: 正常电芯应在2500~4200mV, 未接电芯应在初始值3600mV
      Pack: 应为各节之和
      Temp: 应为室温附近(25±10°C)
      Curr: 无负载时应为0mA(±3mA死区)
      GAIN: 应~377, Offset: 应为小的有符号数
  - 如果电压全为3600mV → I2C通信失败, 检查PB8/PB9接线和上拉电阻
  - 优先看: bq76940.c → BQ76940_ReadAllCells() → BQ_FilterAndUpdateCell()

  步骤4: 检查保护阈值是否生效
  -----------------------------------------------------------------------
  - 单体过压测试: 用电源给某节电芯充到>4200mV
      预期: CHG MOS自动关闭, ov_flag=1
      验证: 发送 status, 检查 "Prot: OV=1"
  - 单体欠压测试: 放电至某节<2800mV
      预期: DSG MOS自动关闭, uv_flag=1
  - 优先看: bq76940.c → BQ_CheckOV() / BQ_CheckUV()
  - 关键变量: g_prot.ov_flag, g_prot.uv_flag

  步骤5: 检查充放电MOS控制是否正常
  -----------------------------------------------------------------------
  - 串口发送: chg on / chg off / dsg on / dsg off / mos on / mos off
  - 每次发送后 status 检查 "CHG: ON/OFF  DSG: ON/OFF"
  - 注意: 如果有保护标志未清, OpenCHG/OpenDSG会先清SYS_STAT
  - 优先看: bq76940.c → BQ76940_OpenCHG() 等6个MOS函数
  - 关键变量: bms_data.chg_on, bms_data.dsg_on

  步骤6: 检查通信是否正常
  -----------------------------------------------------------------------
  - 持续发送 status 命令, 观察数据是否稳定更新
  - 观察 cell_invalid_cnt 和 temp_invalid_cnt 是否持续增长
      如果增长过快 → I2C信号质量差, 检查上拉电阻(建议4.7K)
  - 观察通信健康: 如果连续失败>=5次会自动重初始化
      重初始化后会打印新的GAIN/Offset值
  - 优先看: bq76940.c → BQ76940_CommCheck() → g_bq_ack_fail_continuous

  步骤7: 检查故障触发与恢复是否符合预期
  -----------------------------------------------------------------------
  - 过压: 充到>4200mV → CHG关 → 降到<4100mV → CHG自动恢复
  - 欠压: 放到<2800mV → DSG关 → 升到>3000mV → DSG自动恢复
  - 过流: 电流>5A → 全部MOS关 → 电流降到<4.5A → 自动恢复
  - 过温: 加热>60°C → 全部MOS关 → 降到<55°C → 自动恢复
  - 低温: 降到<0°C → CHG关 → 升到>5°C → CHG自动恢复(放电不受影响)
  - 优先看: bq76940.c → BQ76940_ProtectionCheck()
  - 注意: 所有保护触发时会自动清除均衡(ClearBalance)

  步骤8: 常见失败现象与排查入口
  -----------------------------------------------------------------------
  +---------------------------+-----------------------------------------+
  | 现象                      | 排查入口                                |
  +---------------------------+-----------------------------------------+
  | 串口无任何输出            | 检查波特率115200, PA9/PA10接线           |
  |                           | 检查MX_USART1_UART_Init()               |
  +---------------------------+-----------------------------------------+
  | 所有电压=3600mV           | I2C通信失败, 检查PB8/PB9开漏+上拉       |
  |                           | 检查PA8唤醒脉冲, BQ76940是否焊接正确    |
  |                           | 看 bms_data.adc_gain 是否为365(无校准)  |
  +---------------------------+-----------------------------------------+
  | 电压跳变/不稳定           | 检查滤波参数: BQ_FILTER_WINDOW_SIZE     |
  |                           | 看 cell_invalid_cnt 是否增长快           |
  |                           | 加大I2C_Delay()延时或加大上拉电阻       |
  +---------------------------+-----------------------------------------+
  | 温度显示25°C不变          | NTC未接或接触不良, 检查TS1引脚           |
  |                           | 看 temp_invalid_cnt 和 temp_raw_degC    |
  +---------------------------+-----------------------------------------+
  | MOS无法打开               | SYS_STAT可能有未清标志, 先发 mos on     |
  |                           | 检查g_prot各标志, 保护未释放则无法开MOS  |
  +---------------------------+-----------------------------------------+
  | 系统反复复位              | IWDG超时(~6.5s), 看是否BMS_Task卡死     |
  |                           | 在I2C通信 → 检查总线死锁                |
  |                           | FreeRTOS堆不足 → 减小栈或增大堆         |
  +---------------------------+-----------------------------------------+
  | SOC不变                   | 检查SOC_Update是否被调用                |
  |                           | 检查current_mA是否为0(死区内)           |
  |                           | 检查dt_ms是否合理(应~500)               |
  +---------------------------+-----------------------------------------+



  BMS专项: 保护项详细总结
  -----------------------------------------------------------------------

  (1) 单体过压
      阈值定义: bq76940.h → BQ_OV_THRESHOLD = 4200mV
      恢复阈值: bq76940.h → BQ_OV_RECOVER = 4100mV (回差100mV)
      判断函数: bq76940.c → BQ_CheckOV()
      触发后:   关充电MOS (BQ76940_CloseCHG) + 清均衡 + 清SYS_STAT
      恢复条件: 全部15节 < 4100mV → 自动OpenCHG
      硬件兜底: OV_TRIP寄存器 = 4400mV (BQ76940硬件保护)

  (2) 单体欠压
      阈值定义: bq76940.h → BQ_UV_THRESHOLD = 2800mV
      恢复阈值: bq76940.h → BQ_UV_RECOVER = 3000mV (回差200mV)
      判断函数: bq76940.c → BQ_CheckUV()
      触发后:   关放电MOS (BQ76940_CloseDSG) + 清均衡 + 清SYS_STAT
      恢复条件: 全部15节 > 3000mV → 自动OpenDSG
      硬件兗底: UV_TRIP寄存器 = 2400mV

  (3) 充电过流 / 放电过流 (统一处理)
      阈值定义: bq76940.h → BQ_OC_THRESHOLD = 5000mA
      恢复阈值: bq76940.h → BQ_OC_RECOVER = 4500mA (回差500mA)
      判断函数: bq76940.c → BQ_CheckOC()
      触发后:   关全部MOS (BQ76940_CloseAll) + 清均衡 + 清SYS_STAT
      恢复条件: |I| < 4500mA → 自动OpenAll
      特点:     取绝对值比较, 充放电统一保护

  (4) 短路保护
      阈值定义: bq76940.c → BQ76940_Init() 中 PROTECT1=0xFF
      判断函数: BQ76940硬件自动检测 (非软件)
      触发后:   BQ76940硬件自动关闭DSG FET
      读取方式: BQ76940_ReadAlerts() → g_prot.hw_scd
      软件无额外处理, 完全依赖芯片硬件

  (5) 过温保护
      阈值定义: bq76940.h → BQ_OT_THRESHOLD = 60°C
      恢复阈值: bq76940.h → BQ_OT_RECOVER = 55°C (回差5°C)
      判断函数: bq76940.c → BQ_CheckOT()
      触发后:   关全部MOS (BQ76940_CloseAll) + 清均衡 + 清SYS_STAT
      恢复条件: temp < 55°C → 自动OpenAll

  (6) 低温充电保护
      阈值定义: bq76940.h → BQ_LT_THRESHOLD = 0°C
      恢复阈值: bq76940.h → BQ_LT_RECOVER = 5°C (回差5°C)
      判断函数: bq76940.c → BQ_CheckLT()
      触发后:   关充电MOS (BQ76940_CloseCHG) + 清均衡
      恢复条件: temp > 5°C → 自动OpenCHG
      说明:     仅禁止充电, 放电不受影响
      原理:     锂电池0°C以下充电会导致锂枝晶析出, 严重影响安全

  (7) 均衡相关保护
      所有保护触发时自动清除均衡 (BQ76940_ClearBalance)
      电压采集时暂停均衡 (防止均衡电流影响采样精度)
      无"均衡过热"或"均衡超时"保护



九、芯片 / BMS工程相关编程要点备忘
================================================================================

  9.1 采样时序: BMS_Task每500ms采集一次, ALERT可提前唤醒
      采集顺序: 电压(15节) -> 温度 -> 电流 -> MOS状态
      电压采集前暂停均衡(5ms), 采完恢复

  9.2 数据有效性: 电压2000~4500mV, 温度-40~120度C
      无效不覆盖上次值; 首次上电: 电压=3600mV, 温度=25度C
      跳变限幅: >30mV连续3次才接受; 4点滑动平均+去抖

  9.3 保护阈值(软件): OV>4200关CHG, UV<2800关DSG, OC>5000关ALL, OT>60关ALL, LT<0关CHG
      保护阈值(硬件): OV_TRIP=4400, UV_TRIP=2400 (芯片兗底)
      欠压恢复3000mV(200mV回差), 过流恢复4500mA(500mA回差)

  9.4 MOS时序: 开MOS前必须清SYS_STAT=0xFF; CHG/DSG可独立控制
      通过xI2CMutex保证BMS_Task和UART_Task不冲突

  9.5 均衡: 仅串口手动使能, 无自动均衡策略
      保护触发/初始化/采样期间自动清除均衡
      建议奇偶交替, 避免相邻同时均衡

  9.6 故障恢复: 全部自动恢复, 无锁死/手动恢复模式
      OV恢复: 全部<4100; UV恢复: 全部>3000; OC恢复: |I|<4500; OT恢复: T<55; LT恢复: T>5

  9.7 Flash存储: 0x0800FC00, 20字节, 半字异或校验
      60s检查, 变化>=1%才写; Flash耐久~10000次, 建议外挂EEPROM

  9.8 通信: USART1 115200/8N1, ASCII命令+回车, DMA发送+200ms超时
      I2C写带CRC8(key=7), 读不带CRC; 连续5次失败->重初始化(冷却3s)

  9.9 看门狗: IWDG PSC=64 RLD=4095, 约6.5s; Idle钩子喂狗+WFI省电
      任何任务卡死->Idle不执行->看门狗复位

  9.10 FreeRTOS: BMS_Task(pri=3,500ms), UART_Task(pri=2,50ms), Idle(pri=0)
       堆10KB, 抢占式, Mutex+优先级继承, ALERT用TaskNotify快速唤醒
       中断优先级: TIM3=1, EXTI2/USART1/DMA=6, SysTick/PendSV=15
       configLIBRARY_MAX_SYSCALL_INTERRUPT_PRIORITY=5 (>=5可调FromISR)
