================================================================================
一、系统角色说明
================================================================================

本程序是一个 STM32F407ZGT6 的 Bootloader（引导加载器），驻留在内部 Flash
的 Sector 0（0x08000000，16KB），芯片上电后最先执行。

核心职责：
  1. 检查 EEPROM（W24C02）中的更新标志，判断是否需要 OTA 升级
  2. 提供 5 秒按键窗口：KEY1 触发恢复出厂设置，KEY2 触发进入 CAN 工具
  3. 如需更新：从外置 SPI Flash（W25Q128）读取固件，写入内置 Flash Sector 3
  4. 根据最终状态，跳转到三个目标之一：
     - Sector 1（0x08004000）：出厂默认程序
     - Sector 2（0x08008000）：CAN 更新工具
     - Sector 3（0x0800C000）：用户应用程序

支持的模式：
  +----------+--------+--------------------+------------------+
  | 模式     | 标志值 | 触发方式           | 跳转目标         |
  +----------+--------+--------------------+------------------+
  | 正常启动 | 0x5A   | 默认（无更新请求） | Sector 3 用户APP |
  | OTA 更新 | 0xA5   | APP写EEPROM后复位  | Sector 3 用户APP |
  | 恢复出厂 | 0xFF   | 5s内按KEY1         | Sector 1 默认APP |
  | CAN 工具 | 0xCC   | 5s内按KEY2         | Sector 2 CAN工具 |
  +----------+--------+--------------------+------------------+

外设使用：
  - I2C1（PB8-SCL, PB9-SDA）：读写 W24C02 EEPROM，100kHz
  - SPI1（PB3-SCK, PB4-MISO, PB5-MOSI, PG6-CS）：读 W25Q128 Flash，9MHz
  - USART1（PA9-TX, PA10-RX）：调试串口输出，115200bps
  - GPIO EXTI：KEY1(PC13) 上升沿中断，KEY2(PA0) 上升沿中断

时钟配置：
  HSE=25MHz, PLL: M=25 N=144 P=2 -> SYSCLK=72MHz
  AHB=72MHz, APB1=36MHz, APB2=72MHz, Flash等待=2WS

二、内存 / Flash 划分图
================================================================================

【STM32F407 内部 Flash 布局】

    地址            大小    用途
    +--------------+------+----------------------------------+
    | 0x0800 0000  | 16KB | Sector 0 - Bootloader (本工程)   |  <-- 加载起点
    +--------------+------+----------------------------------+
    | 0x0800 4000  | 16KB | Sector 1 - 出厂默认程序          |  <-- 恢复出厂跳转目标
    +--------------+------+----------------------------------+
    | 0x0800 8000  | 16KB | Sector 2 - CAN 更新工具          |  <-- CAN模式跳转目标
    +--------------+------+----------------------------------+
    | 0x0800 C000  | 16KB | Sector 3 - 用户应用程序          |  <-- OTA/正常跳转目标
    +--------------+------+----------------------------------+
    | 0x0801 0000  | 64KB | Sector 4 - 未使用                |
    | ...          |      |                                  |
    | 0x080E 0000  |128KB | Sector 11- 未使用                |
    +--------------+------+----------------------------------+

    Bootloader 链接脚本（P02_boot_loader.sct）:
    LR_IROM1  0x08000000  0x00004000   （加载区 = Sector 0, 16KB）
    ER_IROM1  0x08000000  0x00004000   （代码执行区）
    RW_IRAM1  0x20000000  0x0001C000   （112KB 主SRAM）
    RW_IRAM2  0x2001C000  0x00004000   （16KB 备用SRAM）


【SRAM 布局】

    地址            大小     用途
    +--------------+--------+-----------------------------+
    | 0x2000 0000  | 112KB  | RW_IRAM1: 全局变量+堆+栈    |
    +--------------+--------+-----------------------------+
    | 0x2001 C000  | 16KB   | RW_IRAM2: 备用              |
    +--------------+--------+-----------------------------+
    | 0x2001 FFFF  |        | SRAM 结束                   |
    +--------------+--------+-----------------------------+

    栈顶合法性校验范围: 0x20000000 ~ 0x2001FFFF


【外置 W25Q128 SPI Flash 布局（16MB）】

    地址               用途
    +-----------------+------------------------------------+
    | 0x00 0000       | 元数据区（8字节）                    |
    |   [0:3] = 程序起始地址（24位，大端序）                  |
    |   [4:7] = 程序长度（字节数，大端序）                    |
    +-----------------+------------------------------------+
    | 0x00 1000       | 最早可存放程序数据的地址              |
    |  ~              |（PROGRAM_START_ADDRESS_MIN）         |
    | 0xFF FFFF       | 16MB 结束                           |
    +-----------------+------------------------------------+

    寻址方式: block(8b) + sector(4b) + page(4b) + addr(8b) = 24bit
    块=64KB, 扇区=4KB, 页=256B


【W24C02 EEPROM 布局（256字节）】

    地址    大小    内容
    +------+------+----------------------------------+
    | 0x10 | 1B   | Bootloader 标志字节               |
    | 0x11 | 2B   | 校验密钥（高字节在前: 0x3C 0x4D） |
    +------+------+----------------------------------+

4.1 树状调用图（带注释）

main()                                                         [Core/Src/main.c]
 |
 +-- HAL_Init()                                    [HAL库初始化，配置SysTick 1ms]
 +-- SystemClock_Config()                 [HSE+PLL -> 72MHz, APB1=36M, APB2=72M]
 +-- MX_GPIO_Init()                                          [Core/Src/gpio.c]
 |    +-- KEY1(PC13) EXTI上升沿中断, 优先级4                 [KEY1 -> 恢复出厂]
 |    +-- KEY2(PA0)  EXTI上升沿中断, 优先级4                 [KEY2 -> CAN工具]
 |    +-- W25Q128_CS(PG6) 推挽输出, 默认高电平               [SPI片选]
 +-- MX_USART1_UART_Init()                                   [Core/Src/usart.c]
 |    +-- 115200/8N1, USART1中断优先级5
 +-- MX_I2C1_Init()                                          [Core/Src/i2c.c]
 |    +-- 100kHz, PB8(SCL)/PB9(SDA)
 +-- MX_SPI1_Init()                                          [Core/Src/spi.c]
 |    +-- Master/8bit/CPOL0/CPHA0/DIV8=9MHz
 |
 +-- App_bootloader_NeedUpdate()              [MDK-ARM/application/App_bootloader.c]
 |    +-- printf("BL Start")                                 [串口打印启动标志]
 |    +-- W24C02_ReadBytes(0x10, data, 3)            [interface/int_w24c02.c]
 |    |    +-- HAL_I2C_Mem_Read(&hi2c1, 0xA0, ...)  [读取1字节标志+2字节密钥]
 |    +-- 校验密钥: key = (data[1]<<8)|data[2]              [期望 0x3C4D]
 |    |    +-- 密钥错误 -> W24C02_WriteBytes() 重置为 {0x5A, 0x3C, 0x4D}
 |    +-- 密钥正确且 data[0]==0xA5 -> App_bootlaoder_status = 0xA5
 |
 +-- App_bootloader_check_factory_reset()           [App_bootloader.c]
 |    +-- HAL_Delay(5000)                                    [5秒按键等待窗口]
 |         （期间 EXTI 中断可改写 App_bootlaoder_status）
 |
 +-- App_bootloader_Update()                        [App_bootloader.c]
 |    +-- [0xA5 更新模式]:
 |    |    +-- W24C02_WriteByte(0x10, 0x5A)           [清除更新标志防重复]
 |    |    +-- App_bootloader_update_from_external_flash()   [static]
 |    |         +-- App_bootloader_check_update_condition()  [static]
 |    |         |    +-- W25Q128_ReadData(0,0,0,0, buf, 8)  [读元数据8字节]
 |    |         |    +-- 解析: program_start_address (字节0-3, 大端)
 |    |         |    +-- 解析: program_length (字节4-7, 大端)
 |    |         |    +-- 校验地址: 0x001000 <= addr <= 0xFFFFFF
 |    |         |    +-- 校验长度: 500 <= len <= 16384
 |    |         +-- App_bootloader_erase_internal_flash_app_area()  [static]
 |    |         |    +-- HAL_FLASH_Unlock()
 |    |         |    +-- HAL_FLASHEx_Erase(Sector3, VoltRange3)
 |    |         |    +-- HAL_FLASH_Lock()
 |    |         +-- 逐页搬运循环:
 |    |              +-- HAL_FLASH_Unlock()
 |    |              +-- while(bytes_written < program_length)
 |    |              |    +-- 计算 read_size (最多256B)
 |    |              |    +-- 拆解24位外置Flash地址 -> block/sector/page/addr
 |    |              |    +-- memset(buf, 0xFF, 256)
 |    |              |    +-- W25Q128_ReadData(b,s,p,a, buf, size)
 |    |              |    |    +-- SPI1: 0x03 + 24bit_addr + 逐字节读
 |    |              |    +-- write_size = (read_size+3) & ~3  [4字节对齐]
 |    |              |    +-- HAL_FLASH_Program(WORD, dest, word) [逐字写入]
 |    |              +-- HAL_FLASH_Lock()
 |    |
 |    +-- [0xFF 恢复出厂]:
 |    |    +-- App_bootloader_erase_internal_flash_app_area()  [擦Sector3]
 |    |    +-- W24C02_WriteBytes(0x10, {0x5A,0x3C,0x4D}, 3)   [重置EEPROM]
 |    |
 |    +-- [0xCC CAN工具]:
 |    |    +-- printf("CAN update mode")   [仅打印, 不做额外操作]
 |    |
 |    +-- [else 正常/0x5A]:
 |         +-- printf("Jump APP")
 |         +-- App_bootloader_JumpToApp()                    [提前跳转]
 |
 +-- App_bootloader_JumpToApp()                     [App_bootloader.c]
      +-- [0xFF] -> Int_bootloader_jump_to_default_app()  [int_bootloader.c]
      +-- [0xCC] -> Int_bootloader_jump_to_can_tool()     [int_bootloader.c]
      +-- [else] -> Int_bootloader_jump_to_app()          [int_bootloader.c]
                     |
                     +-- 校验 MSP: *(0x0800C000) 在 0x20000000~0x2001FFFF
                     +-- 校验 Reset: *(0x0800C004) 在 Sector 范围内
                     +-- NVIC_DisableIRQ(EXTI0/EXTI15_10/USART1)
                     +-- SysTick->CTRL/LOAD/VAL = 0
                     +-- HAL_DeInit()
                     +-- __disable_irq()
                     +-- __set_MSP(app_stack_pointer)
                     +-- SCB->VTOR = 目标Sector起始地址
                     +-- app_entry()  ------------->  [跳转, 不返回]

================================================================================
五、流程图（状态流转）
================================================================================

当前工程采用顺序判断架构（非循环状态机），流程如下：

    +==================+
    |     上电复位      |
    +========|=========+
             |
             v
    +------------------+
    |  HAL_Init()      |
    |  SystemClock     |
    |  GPIO/UART/I2C   |
    |  /SPI Init       |
    +--------|---------+
             |
             v
    +------------------+      密钥错误       +-------------------+
    | NeedUpdate()     |--------------------->| 重置EEPROM为      |
    | 读EEPROM 3字节   |                      | {0x5A,0x3C,0x4D}  |
    | 校验密钥0x3C4D   |                      +--------+----------+
    +--------|---------+                               |
             | 密钥正确                                 |
             v                                         |
    +------------------+                               |
    | flag==0xA5 ?     |---是--> status=0xA5           |
    | 否则 status=0x5A |<------------------------------+
    +--------|---------+
             |
             v
    +------------------+     KEY1中断                KEY2中断
    | check_factory    |  (EXTI15_10)              (EXTI0)
    | _reset()         |     |                        |
    | HAL_Delay(5000)  |     v                        v
    | [5秒按键窗口]    |  status=0xFF             status=0xCC
    +--------|---------+
             |
             v
    +------------------+
    |   Update()       |
    |   检查 status    |
    +--+-+-+-+---------+
       | | | |
       | | | +--[else/0x5A]-----> printf("Jump APP")
       | | |                           |
       | | +----[0xCC]----------> printf("CAN update mode")
       | |                             |
       | +------[0xFF]----------> 擦除Sector3 + 重置EEPROM
       |                               |
       +--------[0xA5]----------> 清EEPROM标志
                                       |
                                       v
                               +----------------+    校验失败
                               | check_update   |-----------> printf + return
                               | _condition()   |             (不擦除不写入)
                               +-------|--------+
                                       | 校验通过
                                       v
                               +----------------+
                               | 擦除 Sector 3  |
                               | 逐页从W25Q128  |
                               | 读取 -> 写入   |
                               | 内置Flash      |
                               +-------|--------+
                                       |
                                       v
                               printf("Update OK")
                                       |
       所有分支汇合                     |
       <--------------------------------+
             |
             v
    +------------------+
    |  JumpToApp()     |
    |  根据 status:    |
    +--+-+-+-----------+
       | | |
       | | +---[0xFF]---> Int_bootloader_jump_to_default_app()
       | |                  -> 校验MSP/Reset -> 跳转 Sector 1
       | |
       | +-----[0xCC]---> Int_bootloader_jump_to_can_tool()
       |                    -> 校验MSP/Reset -> 跳转 Sector 2
       |
       +-------[else]----> Int_bootloader_jump_to_app()
                            -> 校验MSP/Reset -> 跳转 Sector 3
                |
                | 跳转成功: 永不返回
                | 跳转失败(MSP/Reset非法): return 1
                v
    +------------------+
    |   while(1) {}    |
    |  （死循环兜底）   |
    +------------------+

    失败复位路径说明:
    - 本工程跳转失败后进入 while(1) 死循环, 需手动复位
    - 无自动 NVIC_SystemReset() 调用
    - 当前启用策略: 5秒定时按键检测（非超时自动跳转）

================================================================================
八、实际使用步骤
================================================================================

8.1 最小可跑通步骤
--------------------------------------------------------------------------------

1) 编译下载 Bootloader
   - 打开 MDK-ARM/P02_boot_loader.uvprojx
   - 编译(Build)确认无错误
   - 使用 ST-Link/J-Link 下载到 Sector 0 (0x08000000)
   - 确认链接脚本 .sct 中 ROM 起始=0x08000000, 大小=0x4000

2) 准备目标 APP 固件
   - APP 工程的链接地址必须设为 0x0800C000 (Sector 3)
   - APP 的 VTOR 必须重定向到 0x0800C000
   - APP 编译后获取 .bin 文件

3) 将 APP 固件写入 W25Q128
   - 在地址 0x000000 写入 8 字节元数据(大端序):
     [0:3] = 程序在 W25Q128 中的起始地址 (如 0x00001000)
     [4:7] = 程序长度 (如 0x00002000 = 8192字节)
   - 在 W25Q128 对应起始地址写入 APP 的 .bin 数据
   - 可通过上位机/其他工具预先烧写 W25Q128

4) 设置 EEPROM 更新标志
   - 向 EEPROM 0x10 写入 0xA5 (需要更新)
   - 向 EEPROM 0x11-0x12 写入 0x3C 0x4D (校验密钥)
   - 通常由运行中的 APP 写入后执行软复位触发

5) 复位芯片, 观察串口
   - 串口参数: 115200, 8N1, 无流控
   - 预期输出:
     BL Start
     Updating...
     Update OK
   - 然后 Bootloader 跳转到 Sector 3 的 APP

6) 跳转验证
   - APP 应能正常运行
   - 可在 APP 中通过串口输出标识信息验证跳转成功
   - 用调试器检查 SCB->VTOR 是否为 0x0800C000


8.2 恢复出厂设置步骤
--------------------------------------------------------------------------------
   1) 芯片复位
   2) 在 5 秒内按下 KEY1 (PC13)
   3) 串口输出: "Factory reset" -> "Factory reset OK"
   4) 自动跳转到 Sector 1 (0x08004000) 的默认程序
   5) 前提: Sector 1 必须预先烧写了有效的默认程序


8.3 进入 CAN 工具模式
--------------------------------------------------------------------------------
   1) 芯片复位
   2) 在 5 秒内按下 KEY2 (PA0)
   3) 串口输出: "CAN update mode"
   4) 自动跳转到 Sector 2 (0x08008000) 的 CAN 工具程序
   5) 前提: Sector 2 必须预先烧写了有效的 CAN 工具程序


8.4 常见失败现象与排查入口
--------------------------------------------------------------------------------
+------------------------------+----------------------------------------------+
| 现象                         | 排查方向                                     |
+------------------------------+----------------------------------------------+
| 串口无任何输出               | 1. 检查USART1接线(PA9-TX, PA10-RX)           |
|                              | 2. 确认波特率115200                           |
|                              | 3. 检查HSE晶振是否25MHz                       |
|                              | 4. 确认Bootloader已下载到Sector 0             |
+------------------------------+----------------------------------------------+
| "BL Start" 后直接            | EEPROM标志不是0xA5, 且5秒内未按键             |
| "Jump APP" 但APP未运行       | 检查Sector 3是否有有效程序                    |
|                              | 调试器查看0x0800C000处是否为合法MSP           |
+------------------------------+----------------------------------------------+
| "Invalid addr" 或            | W25Q128元数据校验失败                         |
| "Invalid len"                | 检查W25Q128地址0x000000处的8字节元数据         |
|                              | 地址需在0x001000~0xFFFFFF                     |
|                              | 长度需在500~16384                             |
+------------------------------+----------------------------------------------+
| "Erase err"                  | 内部Flash擦除失败                             |
|                              | 检查VoltageRange是否匹配供电电压              |
|                              | 检查Flash是否被读保护锁定                     |
+------------------------------+----------------------------------------------+
| "Write err"                  | Flash编程失败                                 |
|                              | 目标地址未对齐或未擦除                        |
|                              | 检查flash_dest是否越过Sector 3边界            |
+------------------------------+----------------------------------------------+
| "Check fail"                 | check_update_condition() 返回非0              |
|                              | 不会擦除不会写入, 直接跳过更新                |
+------------------------------+----------------------------------------------+
| 跳转后 HardFault             | 1. APP链接地址是否为0x0800C000                |
|                              | 2. APP是否设置了SCB->VTOR=0x0800C000         |
|                              | 3. APP栈大小是否合理                          |
|                              | 4. Bootloader是否正确清理了中断               |
+------------------------------+----------------------------------------------+
| KEY1/KEY2 按了没反应          | 1. 检查接线和GPIO配置(上升沿+下拉)            |
|                              | 2. 检查EXTI中断是否启用                       |
|                              | 3. 确认按键产生了上升沿                       |
+------------------------------+----------------------------------------------+


================================================================================
九、芯片相关的编程要点备忘
================================================================================

9.1 VTOR 重定位时机
--------------------------------------------------------------------------------
  - 必须在跳转前设置 SCB->VTOR = 目标APP起始地址
  - 本工程在 __disable_irq() 之后, app_entry() 之前设置
  - APP 自身也建议在 SystemInit() 或 main() 最开始再次确认 VTOR
  - F407 的 VTOR 对齐要求: 向量表大小向上取2的幂次再对齐
    (本工程 0x4000/0x8000/0xC000 均满足)


9.2 跳转前去初始化与中断清理
--------------------------------------------------------------------------------
  本工程清理顺序 (三个跳转函数一致):
  1) NVIC_DisableIRQ(EXTI0_IRQn)       -- 禁用KEY2外部中断
  2) NVIC_DisableIRQ(EXTI15_10_IRQn)   -- 禁用KEY1外部中断
  3) NVIC_DisableIRQ(USART1_IRQn)      -- 禁用USART1中断
  4) SysTick->CTRL/LOAD/VAL = 0        -- 关闭SysTick
  5) HAL_DeInit()                      -- 注销HAL外设
  6) __disable_irq()                   -- 全局关中断(PRIMASK=1)
  7) __set_MSP(app_stack_pointer)      -- 设置新栈顶
  8) SCB->VTOR = 目标地址              -- 重定向中断向量表
  9) app_entry()                       -- 跳转(不返回)

  注意: HAL_DeInit() 不会清除NVIC中断使能位, 必须手动NVIC_DisableIRQ。
  否则跳转后引脚状态变化会触发旧中断向量, 导致 HardFault。


9.3 MSP / Reset 向量合法性检查
--------------------------------------------------------------------------------
  跳转前必须校验:
  1) MSP (目标地址+0处的值) 在 SRAM 范围内:
     0x20000000 <= MSP <= 0x2001FFFF
  2) Reset Handler (目标地址+4处的值) 在目标 Sector 范围内:
     - Sector 1: 0x08004000 ~ 0x08008000
     - Sector 2: 0x08008000 ~ 0x0800C000
     - Sector 3: 0x0800C000 ~ 0x08010000
  校验失败返回1, 不跳转。
  典型失败: 目标Sector未烧写(全0xFF), MSP=0xFFFFFFFF。


9.4 Flash 扇区边界 / 越界保护
--------------------------------------------------------------------------------
  STM32F407 前4个扇区均为 16KB:
  - Sector 0: 0x08000000~0x08003FFF (Bootloader)
  - Sector 1: 0x08004000~0x08007FFF (默认APP)
  - Sector 2: 0x08008000~0x0800BFFF (CAN工具)
  - Sector 3: 0x0800C000~0x0800FFFF (用户APP)

  写入循环仅以 program_length 为终止条件, 未检查 flash_dest 越界。
  PROGRAM_LENGTH_MAX=16384=Sector 3大小, 理论不越界。
  但4字节对齐可能多写几字节, 建议增加边界检查。


9.5 校验机制
--------------------------------------------------------------------------------
  当前校验层级:
  1) EEPROM密钥校验: 0x3C4D, 防止误触发
  2) 外置Flash元数据校验: 地址范围+长度范围
  3) 跳转前校验: MSP范围+Reset Handler范围

  【CRC校验说明】
  - CRC/完整性校验已在其他分区(APP端或上位机写入W25Q128时)完成
  - Bootloader本身不重复做CRC校验, 属于合理的职责分离设计
  - 写入W25Q128前由上位机/APP保证数据完整性


9.6 其他注意事项
--------------------------------------------------------------------------------
  - W24C02页写入后需5ms延时(芯片内部擦写周期)
  - W25Q128写入/擦除前必须发0x06(Write Enable)
  - W25Q128 0x03读命令限速~33MHz(本工程9MHz, 无问题)
  - printf需确保usart.c中fputc重定向(或使用MicroLib)
  - W25Q128_ReadData32()用0x03命令发4字节地址, 与标准0x03
    (3字节地址)不符, 建议确认或改用0x13命令
